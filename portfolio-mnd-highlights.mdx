# React Highlightable Datasheet — 포트폴리오 하이라이트

엑셀형 데이터 그리드와 줄 단위 텍스트 뷰어를 한 패키지로 묶은 **react-highlightable-datasheet** 프로젝트에서, 포트폴리오·이력서·면접에서 **자랑할 만한 부분**을 정리한 문서입니다.

---

## 0. 공개 범위 (보안)

실제 과제와 연결된 구현이지만, **보안·비공개 정책**상 **기능 목적·업무 맥락·사용 시나리오**는 공개 문서에 적지 않습니다. 아래와 사이트 포스트에는 **기술 요지**(구조, 가상화, 상태 모델 등)만 남겼고, 코드는 **비공개 원본을 재구성한 발췌**입니다.

**포트폴리오 사이트 상세 페이지:** [스프레드시트형 그리드·텍스트 뷰어](https://junuuon.pages.dev/projects/mnd-excel-viewer) — 긴 설명·코드 발췌는 여기에 있습니다. (저장소: `src/lib/posts/mnd-excel-viewer.ko.svx` / `.en.svx`)

---

## 1. 제품·문제 정의

**엑셀형 그리드 + 줄 단위 텍스트 뷰어**를 한 패키지로 제공하고, **드래그 선택·하이라이트·(협업용) 소켓 훅**까지 포함했습니다. “일반 테이블”이 아니라 **선택 영역 좌표 모델·시각적 피드백·외부 시스템 연동**까지 설계한 **도메인이 분명한 UI 라이브러리**로 설명할 수 있습니다.

---

## 2. 프레젠테이션 / 컨테이너 분리

`ExcelViewer`·`TextViewer`는 상태·이벤트·Popover를 담당하고, `*ViewerBase`는 **표 렌더링·가상 스크롤**에 집중합니다. 재사용·테스트·스토리북 분리에 유리한 구조이며, 포트폴리오에서 **관심사 분리** 근거로 쓰기 좋습니다.

---

## 3. 엑셀형 UX (react-table + 레이아웃)

`react-table` v7의 **flex 레이아웃, 컬럼 리사이즈, 정렬**을 함께 사용합니다. 스프레드시트에 가까운 헤더/바디 구조를 라이브러리 조합으로 구현했다고 정리할 수 있습니다.

---

## 4. 컬럼 라벨 알고리즘 (A, B, …, Z, AA, …)

단순 `String.fromCharCode` 한 번이 아니라 **엑셀식 열 이름**을 생성하는 로직을 직접 구현했습니다. **도메인에 맞는 작은 알고리즘** 예시입니다. (코드 블록은 [상세 페이지](https://junuuon.pages.dev/projects/mnd-excel-viewer) 참고.)

---

## 5. 2차원 영역 선택 + Immer로 드래그 업데이트

그리드는 `column:row` 문자열 ID로 앵커를 잡고, 드래그 중 **`min` / `max`로 직사각형 영역**을 유지합니다. 불변 업데이트는 **Immer `produce`**로 처리해, **복잡한 상태를 안전하게 갱신**했다는 면접 답변 근거가 됩니다. **Ctrl + 드래그로 다중 영역**(`count` 증가)까지 이어지는 점은 “스프레드시트 느낌” 스토리와 연결할 수 있습니다. (코드 발췌는 상세 페이지 동일.)

---

## 6. 텍스트 뷰어: 줄 범위·역방향 드래그 보정

줄 번호 기준 선택에서, 마우스 업 시 **start/end가 뒤집힌 경우 정규화**하는 처리가 있습니다. UX 디테일을 신경 썼다는 근거가 됩니다.

---

## 7. 성능: 가상화를 뷰 특성에 맞게 이원화

- **그리드**: `react-window`의 `VariableSizeList` + `InfiniteLoader` + `AutoSizer`로 **행 단위 가상 스크롤** (대량 행 대비).
- **텍스트**: `react-virtualized`의 **`CellMeasurerCache`**로 줄 높이를 측정·캐시 — **가변 높이 줄**에 맞춘 선택이 포인트입니다.

한 가상화 라이브러리만 쓴 것이 아니라 **뷰별로 도구를 골랐다**고 말하면 설득력이 있습니다.

---

## 8. 협업 확장 포인트 (Socket.IO)

- 하이라이트 제거 시 `socket.emit("removeHighlight", { fileId, highlightId })`로 **서버 계약**을 명시했습니다.
- `App.tsx`의 **`createMockSocket`**으로 실서버 없이도 동일 인터페이스로 데모 가능 — **의존성 주입·데모/테스트 전략**으로 짧게 언급하기 좋습니다.

---

## 9. UI·접근성·안정성

- **styled-components**로 선택/하이라이트/오버레이 상태를 조합하고, `::selection` 투명 처리 등 **브라우저 기본 선택과의 충돌 완화** 스타일이 있습니다.
- **sticky 헤더**, 스크롤바 테마 등 그리드 도구 느낌을 살린 부분이 있습니다.
- **ErrorBoundary**로 런타임 오류 시 복구 UI를 제공합니다.

---

## 10. TypeScript

`ExcelViewerBase`에서 `HeaderGroup`에 **리사이저·정렬 관련 옵셔널 필드**를 확장해 타이핑한 부분은, **서드파티 훅과 타입을 맞추는 경험**으로 짧게 넣을 수 있습니다.

---

## 11. 기술 위주로 말하기 (추가)

목적·도메인은 말하지 않아도, 아래는 **구현 관점**에서 짧게 붙이기 좋습니다.

- **허용된 스택·런타임 제약** 안에서 가상화·상태 관리 조합을 고른 근거(면접 시 맥락은 구두로만).
- **그리드 vs 텍스트**에 서로 다른 가상화를 쓴 이유(고정 행 높이 vs 가변 줄 높이).
- **소켓 형태의 API + mock**으로 백엔드 없이도 UI·이벤트 흐름을 검증했다는 점.

---

## 포트폴리오용 한두 문장 예시

> React 기반 **엑셀형 데이터 그리드·줄 단위 텍스트 뷰어**를 구현했고, **2D 영역 선택·다중 선택(Immer)·가상 스크롤(react-window / react-virtualized)**으로 대용량 행·가변 높이 줄을 다뤘습니다. **Socket.IO 형태의 이벤트 계약·mock**으로 앱 셸과 런타임 의존성을 분리했습니다.
