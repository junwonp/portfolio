# KFTC 프로모션 및 리워드 플랫폼

사용자의 적극적인 참여를 유도하고 보상을 제공하는 다목적 프로모션 플랫폼 프론트엔드 프로젝트입니다.

## 📝 프로젝트 배경 및 개요

- **프로젝트 성격**: 금융결제원 외주 개발 프로젝트
- **주요 목표**: 금융결제원의 금융 서비스 이용자들을 위한 챌린지 및 리워드 생태계 구축
- **핵심 성과**:
  - **단기 집중 개발**: 촉박한 일정 속에서도 분석-설계-구현 단계를 효율적으로 운영하여 정해진 기한 내에 성공적으로 런칭했습니다.
  - **서비스 안정성 확보**: 금융 관련 서비스인 만큼 높은 신뢰도가 요구되었으며, 철저한 타입 정의와 에러 핸들링, 그리고 Mock 서버를 활용한 극한의 상황 테스트를 통해 서비스 안정성을 극대화했습니다.
  - **확장성 있는 구조**: 향후 추가될 다양한 형태의 금융 프로모션에 즉각 대응할 수 있도록 비즈니스 로직을 모듈화했습니다.

## 🎯 주요 성과 및 기술적 강점

### 1. 모던 프론트엔드 스택과 견고한 아키텍처 설계

- **최신 기술 스택 도입**: React 19, TypeScript, Vite, TailwindCSS v4를 사용하여 높은 생산성과 빠른 빌드 환경을 구축했습니다.
- **관심사 분리 (SoC)**: `api`, `components`, `hooks`, `context`, `lib` 등으로 폴더 구조를 명확하게 분리하여 코드의 유지보수성과 확장성을 극대화했습니다.
- **안전한 폼 상태 관리**: 회원가입 및 프로필 수정과 같은 복잡한 폼 처리 로직을 `react-hook-form`과 `zod`를 결합하여 선언적이고 타입 안전하게 검증했습니다. 불필요한 리렌더링을 방지하여 성능 최적화도 함께 달성했습니다.

### 2. 상태 관리 및 오프라인 캐싱 고도화

- **서버 상태의 선언적 관리**: `@tanstack/react-query`를 도입하여 복잡한 비동기 상태 관리를 단순화했습니다.
- **IndexedDB 기반 캐시 영속성 구현**: `@tanstack/react-query-persist-client`와 `@tanstack/query-async-storage-persister`, 그리고 `idb-keyval` 라이브러리를 통합하여 클라이언트 사이드 캐싱 구조를 구축했습니다. 이를 통해 네트워크 환경이 불안정한 상태에서도 빠르고 안정적인 오프라인 경험을 제공하며 초기 로딩(FCP) 속도를 크게 개선했습니다.

### 3. Mock 서버 구축을 통한 개발 경험(DX) 향상 및 병렬 개발

- **백엔드 의존성 완화**: Express 기반의 커스텀 Mock 서버(`mock-server`)를 자체적으로 구축하여, 백엔드 API 완성을 기다리지 않고 프론트엔드 개발을 독립적으로 진행했습니다.
- **정교한 상태 및 네트워크 시뮬레이션**: 단순한 JSON 반환을 넘어, 세션 쿠키 기반의 인증 처리, 상태 유지, 페이지네이션, `MOCK_DELAY`를 통한 실제 네트워크 지연 시뮬레이션 등 실서버와 유사한 환경을 구현하여 프론트엔드 로직의 안정성을 높였습니다.

### 4. 복잡한 비즈니스 로직 추상화 및 사용자 경험(UX) 최적화

- **복합 프로모션 로직 처리**: 단순 출석 체크뿐만 아니라 누적 참여, 연속 참여 등 조건이 까다로운 챌린지 비즈니스 로직과 티켓 추첨, 포인트 만료 로직을 파악하기 쉬운 커스텀 훅(Custom Hook)으로 추상화하여 관리했습니다.
- **컴포넌트 주도 개발 (CDD)**: Storybook을 활용해 UI 컴포넌트를 독립된 환경에서 시각적으로 개발 및 테스트했으며, `@storybook/addon-a11y`를 적용하여 웹 접근성(A11y) 표준을 고려한 마크업을 작성했습니다.
- **마이크로 인터랙션 강화**: `@lottiefiles/dotlottie-react`를 도입해 가볍고 부드러운 Lottie 애니메이션(티켓 추첨 룰렛, 출석 도장 등)을 구현하여 사용자에게 직관적이고 긍정적인 피드백을 제공했습니다.
